A group of buttons for multiple choice — single answer, or multi-select with a submit button.

```json
{
  "type": "Choices",
  "props": {
    "event": "answer",
    "multiSelect": false,     // false: tap submits immediately. true: select many, then submit.
    "choices": [
      // Each choice is a Button. See Button for per-choice styling, keys, etc.
      { "label": "A: Helium",  "value": "A", "keys": ["A", "1"] },
      { "label": "B: Neon",    "value": "B", "keys": ["B", "2"] },
      { "label": "C: Krypton", "value": "C", "keys": ["C", "3"] },
      { "label": "D: Boron",   "value": "D", "keys": ["D", "4"] }
    ],
    "submit": {                // shown only when multiSelect is true
      "label": "Submit",       // "Enter" also submits
      "style": { "margin": "50px 0px" }
    },
    "style": {
      "grid": false,           // false: stacked. true: grid layout.
      "gridColumns": 2,
      "gridRowHeight": "1fr",
      "gridGap": "10px"
    }
  }
}
```

| Prop | Type | Notes |
| --- | --- | --- |
| `event` | string | The event name your host receives. |
| `multiSelect` | boolean | Default `false`. See submit behavior below. |
| `choices` | object[] | Each choice is a [Button](/components/button) (`label`, `value`, optional `keys`, `style`). |
| `submit` | object | Submit button `label` and `style`. Only shown when `multiSelect` is `true`. |
| `style` | object | Layout: `grid` (bool), `gridColumns`, `gridRowHeight`, `gridGap`. |

**Sends:** `change` on each selection. When `multiSelect` is `false`, tapping a choice immediately sends `msg` with `value` set to that choice's `value`. When `multiSelect` is `true`, the player toggles choices and taps submit (or presses **Enter**) to send `msg` with `value` as an **array** of selected values.
