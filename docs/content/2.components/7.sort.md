---
title: Sort
description: Players drag options into an order, then submit.
---

Players drag all the options into an order, then submit. Configured like [Choices](/components/choices), with two differences: there's no `multiSelect` (you always sort the full list), and choices don't take `keys`.

```json
{
  "type": "Sort",
  "props": {
    "event": "ranking",
    "choices": [
      {
        "label": "🍕 Pizza",
        "value": "pizza"
      },
      {
        "label": "🌮 Tacos",
        "value": "tacos"
      },
      {
        "label": "🍔 Burger",
        "value": "burger"
      }
    ],
    "submit": { "label": "Lock it in" },
    "style": { "gridGap": "10px" }
  }
}
```

| Prop | Type | Notes |
| --- | --- | --- |
| `event` | string | The event name your host receives. |
| `choices` | object[] | Each option's `label`, `value`, and optional `style`. No `keys`. |
| `submit` | object | Submit button `label` and `style`. |
| `style` | object | Layout: `grid`, `gridColumns`, `gridRowHeight`, `gridGap`. |

**Sends:** `msg` on submit, with `value` set to the array of `value`s **in the player's chosen order** (top to bottom).
