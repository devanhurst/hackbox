Components are the building blocks of a player's screen. You list them in a member state's `ui.main.components`, and Hackbox renders them top to bottom:

```json
{
  "ui": {
    "main": {
      "components": [
        {
          "type": "Text",
          "props": { "text": "Round 1" }
        },
        {
          "type": "Button",
          "props": {
            "event": "buzz",
            "label": "Buzz in!",
            "value": "BUZZ"
          }
        }
      ]
    }
  }
}
```

Each component is `{ "type": <ComponentName>, "props": { ... } }`. Browse the sidebar for a page on every component.

## How components talk back

Interactive components send your host a `msg` (on submit) or `change` (while editing) event — see [Core concepts](/basics/concepts#events-from-players). Every emission carries the same three fields in its `message`:

- **`event`** — the component's `event` prop, so you can route it.
- **`value`** — the component's value. The shape depends on the component.
- **`ms`** — milliseconds since the component appeared on screen — a built-in reaction timer.

| Component | Sends `change` | Sends `msg` | `value` shape |
| --- | :---: | :---: | --- |
| [Text](/components/text) | — | — | _(display only)_ |
| [Button](/components/button) | — | on tap | the `value` prop |
| [Buzzer](/components/buzzer) | — | on tap | `null` (use `ms` for reaction time) |
| [TextInput](/components/text-input) | while typing | on submit | the entered string |
| [Choices](/components/choices) | on each selection | on submit | a string, or an array if `multiSelect` |
| [Sort](/components/sort) | — | on submit | the values in the player's chosen order |
| [Range](/components/range) | while dragging | on submit | the number, as a string |

## The `style` object

Most components accept a `style` object that's applied directly to the component. It takes **any standard CSS property**, written in camelCase — there's no fixed list and no validation, so `boxShadow`, `textTransform`, `letterSpacing`, `backdropFilter`, and anything else valid all work:

```json
"style": {
  "color": "black",
  "textAlign": "center",
  "background": "white",
  "border": "4px solid black",
  "borderRadius": "10px",
  "boxShadow": "2px 2px #0003",
  "width": "auto",
  "fontSize": "16px",
  "padding": "10px",
  "margin": "10px 0px",
  "fontFamily": "sans-serif"
}
```

`fontFamily` accepts **any [Google Fonts](https://fonts.google.com) family name** — Hackbox loads it automatically. See [Theming & layout](/advanced/theming#fonts).

::callout{icon="i-lucide-info" color="warning"}
**Deprecated — `align`.** Older payloads used an `align` key to set text/content alignment. It still works (it maps to `textAlign` + `justifyContent`), but it's deprecated — prefer the standard **`textAlign`**. This is unrelated to [`ui.main.align`](/basics/concepts#member-state), which positions a whole component within the screen and is not deprecated.
::

To avoid repeating the same `style` on every component, define it once with [Presets](/advanced/presets).
