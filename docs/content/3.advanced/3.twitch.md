---
title: Twitch auth
description: Tie players to their Twitch identity for stream games.
---

Hackbox can require players to sign in with Twitch before they join. This is handy for streamed games — you get each player's Twitch handle and avatar, and you can trust that a player is who their name says they are.

You don't implement any OAuth yourself. Hackbox's player client runs the entire Twitch login flow; your host just opts in and reads the result.

## Requiring Twitch

Set `twitchRequired: true` when you [create the room](/basics/getting-started#1-create-a-room):

```json
// POST https://hackbox.ca/api/rooms
{
  "hostId": "11111111-2222-3333-4444-555555555555",
  "twitchRequired": true
}
```

Now, when someone opens the room on their phone, Hackbox's client prompts them to log in with Twitch. A player who hasn't authenticated is **refused entry** — the relay rejects the connection with an [`error`](/advanced/errors-and-lifecycle#error-frames):

> Please log in with Twitch before joining this room.

So by the time a player shows up in your roster, they're guaranteed to have a verified Twitch identity.

## Reading a player's Twitch identity

Authenticated players carry a `twitchData` object in every [`state.host`](/basics/concepts#events-from-players) roster entry:

```json
{
  "type": "state.host",
  "payload": {
    "members": {
      "aaaa...": {
        "id": "aaaa...",
        "name": "HEYDEVDEV",
        "online": true,
        "twitchData": {
          "id": "56475817",
          "username": "heydevdev",
          "photo": "https://static-cdn.jtvnw.net/jtv_user_pictures/....png"
        }
      }
    }
  }
}
```

| Field | What it is |
| --- | --- |
| `twitchData.id` | The player's numeric Twitch user ID — a stable key to recognize returning viewers. |
| `twitchData.username` | Their Twitch display name. |
| `twitchData.photo` | URL of their Twitch profile image — drop it straight into your host UI. |

Use `twitchData.id` (not the display name) if you want to recognize the same viewer across sessions.

## Notes

- `twitchData` is only present when the player authenticated. In a room created **without** `twitchRequired`, players join anonymously and have no `twitchData`.
- Hackbox verifies each token against Twitch server-side; an invalid or expired token is treated as "not logged in." Your host never sees the raw token.
