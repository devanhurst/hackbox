---
title: Theming & layout
description: Style a player's screen — colors, fonts, headers, and alignment.
---

Beyond the [components](/components/overview) themselves, a member state controls the look of the whole screen through `theme`, the header, and alignment. Like everything in a member state, these are part of a [full replacement](/basics/concepts#state-is-a-full-replacement) — include them in every `member.update` you want them to apply to.

## The theme object

```json
{
  "theme": {
    "header": {
      "color": "#EEE",            // header text color
      "background": "#222",        // header bar color
      "fontFamily": "Fredoka One"  // header font
    },
    "main": {
      "background": "#111"         // the screen background behind your components
    }
  }
}
```

These are the screen-level colors and fonts. Individual components carry their own `style` (see [Components](/components/overview#the-style-object)); the `theme` sets the canvas they sit on.

## Header and alignment

The header is a bar at the top of the player's screen. Its text lives in `ui.header.text`; its colors and font come from `theme.header`:

```json
{
  "ui": {
    "header": { "text": "Round 1 — 0:30 left" },
    "main": {
      "align": "center",   // "start" | "center" | "end"
      "components": [ /* ... */ ]
    }
  }
}
```

`ui.main.align` sets how the components sit vertically in the available space:

- **`start`** — top (the default).
- **`center`** — vertically centered.
- **`end`** — bottom (handy for keeping a primary button near the player's thumb).

## Fonts

Any `fontFamily` value — whether in `theme` or a component's `style` — may be **any [Google Fonts](https://fonts.google.com) family name**. Hackbox scans every state you send for font names and loads them from Google Fonts automatically, so you can write:

```json
"fontFamily": "Press Start 2P"
```

and it just works on the player's device. The default font is **`Fredoka One`**.

## Defaults

If you omit them, theme values fall back to Hackbox's dark default palette:

| Key | Default |
| --- | --- |
| `theme.header.color` | `#EEE` |
| `theme.header.background` | `#222` |
| `theme.header.fontFamily` | `Fredoka One` |
| `theme.main.background` | `#111` |
| `ui.header.text` | `""` (empty) |
| `ui.main.align` | `start` |

## A fully-themed example

```json
{
  "type": "member.update",
  "payload": {
    "to": "aaaa...",
    "data": {
      "theme": {
        "header": { "color": "#1a1a2e", "background": "#e94560", "fontFamily": "Bungee" },
        "main": { "background": "#16213e" }
      },
      "ui": {
        "header": { "text": "QUIZ NIGHT" },
        "main": {
          "align": "center",
          "components": [
            { "type": "Text", "props": { "text": "What's the capital of France?", "style": { "color": "#EEE", "background": "transparent", "border": "none", "fontFamily": "Bungee", "fontSize": "1.5rem", "textAlign": "center" } } },
            { "type": "Button", "props": { "event": "answer", "label": "Paris", "value": "paris", "style": { "fontFamily": "Bungee" } } }
          ]
        }
      }
    }
  }
}
```

Repeating the same `style`/`fontFamily` on every component gets verbose fast — [Presets](/advanced/presets) let you define it once and reuse it.
