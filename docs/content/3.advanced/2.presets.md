---
title: Presets
description: Define a component template once and reuse it across a player's screen.
---

Most games reuse the same styled component over and over — four answer buttons that look identical, a heading style repeated every round. **Presets** let you define that template once and reference it by name, so each use only specifies what's different.

## How presets work

Add a `presets` object to a member state. Each key is a name; each value is a normal component definition:

```json
{
  "presets": {
    "AnswerButton": {
      "type": "Button",
      "props": {
        "event": "answer",
        "style": {
          "background": "#e94560",
          "color": "#fff",
          "fontFamily": "Bungee",
          "borderRadius": "12px",
          "margin": "8px 0"
        }
      }
    }
  }
}
```

Then, in `ui.main.components`, reference the preset by using its **name as the component `type`**:

```json
{
  "type": "AnswerButton",
  "props": {
    "label": "Paris",
    "value": "paris"
  }
}
```

When Hackbox renders the screen, it expands that into the preset's real component, merging props:

- The rendered `type` becomes the preset's `type` (`Button` here).
- Props are merged as `{ ...preset.props, ...component.props }` — **the component's own props win** on conflict.

So the example above renders as a `Button` with the preset's styling and event, plus this instance's `label` and `value`.

## Full example

```json
{
  "type": "member.update",
  "payload": {
    "to": "aaaa...",
    "data": {
      "presets": {
        "AnswerButton": {
          "type": "Button",
          "props": {
            "event": "answer",
            "style": {
              "background": "#e94560",
              "color": "#fff",
              "fontFamily": "Bungee",
              "borderRadius": "12px",
              "margin": "8px 0"
            }
          }
        }
      },
      "ui": {
        "main": {
          "components": [
            {
              "type": "AnswerButton",
              "props": {
                "label": "A: Paris",
                "value": "A"
              }
            },
            {
              "type": "AnswerButton",
              "props": {
                "label": "B: London",
                "value": "B"
              }
            },
            {
              "type": "AnswerButton",
              "props": {
                "label": "C: Berlin",
                "value": "C"
              }
            },
            {
              "type": "AnswerButton",
              "props": {
                "label": "D: Madrid",
                "value": "D"
              }
            }
          ]
        }
      }
    }
  }
}
```

Four identically-styled buttons, and the styling lives in exactly one place.

## Notes

- `presets` is part of the member state, so it travels in the same `data` as `theme` and `ui`. Because state is a [full replacement](/basics/concepts#state-is-a-full-replacement), include the `presets` you rely on in each `member.update` that uses them.
- A component only expands if its `type` matches a preset key. Built-in types (`Button`, `Text`, …) render as themselves — avoid naming a preset after a built-in unless you mean to shadow it.
- Override any preset prop per instance by setting it on the component — including nested `style` keys (the merge is shallow at the props level, so a component's `style` replaces the preset's `style` wholesale rather than deep-merging).
