This walkthrough ties everything together by building a multi-round trivia game: players answer multiple-choice questions on their phones, get instant feedback, and watch their score climb. The host code is shown in language-agnostic pseudocode — translate the WebSocket calls to your stack of choice.

By the end you'll have used [components](/components/overview), [theming](/advanced/theming), [presets](/advanced/presets), and both the `msg` and `change` events.

## Set up the room

Create a room and connect as the host, exactly as in [Getting started](/basics/getting-started):

```
roomCode = POST https://hackbox.ca/api/rooms { hostId }   → "HKBX"
socket   = connect wss://hackbox.ca/r/HKBX?userId=<hostId>
every 25s: socket.send("ping")
```

Keep a little game state in your host:

```
players = {}            // userId → { name, score }
currentQuestion = null  // the question on screen
answers = {}            // userId → their submitted choice this round
```

Track who's in the room from `state.host`:

```
on "state.host" (payload):
  for id, member in payload.members:
    if id not in players:
      players[id] = { name: member.name, score: 0 }
```

## Define a look once, with presets and a theme

Rather than restyling every screen, define an answer-button preset and a theme you'll reuse all game. We'll include these in every `member.update`.

```json
{
  "theme": {
    "header": {
      "color": "#fff",
      "background": "#e94560",
      "fontFamily": "Bungee"
    },
    "main": { "background": "#16213e" }
  },
  "presets": {
    "Answer": {
      "type": "Button",
      "props": {
        "event": "answer",
        "style": {
          "background": "#0f3460",
          "color": "#fff",
          "fontFamily": "Bungee",
          "borderRadius": "12px",
          "margin": "8px 0",
          "width": "100%"
        }
      }
    }
  }
}
```

## 1. Ask the question

Send every player the question text and four answer buttons. We use the `Answer` preset, so each button only needs its `label` and `value`:

```json
{
  "type": "member.update",
  "payload": {
    "to": ["aaaa...", "bbbb..."],
    "data": {
      "theme": {
        "header": {
          "color": "#fff",
          "background": "#e94560",
          "fontFamily": "Bungee"
        },
        "main": { "background": "#16213e" }
      },
      "presets": {
        "Answer": {
          "type": "Button",
          "props": {
            "event": "answer",
            "style": {
              "background": "#0f3460",
              "color": "#fff",
              "fontFamily": "Bungee",
              "borderRadius": "12px",
              "margin": "8px 0",
              "width": "100%"
            }
          }
        }
      },
      "ui": {
        "header": { "text": "Question 1" },
        "main": {
          "align": "center",
          "components": [
            {
              "type": "Text",
              "props": {
                "text": "Which planet is the **Red Planet**?",
                "style": {
                  "color": "#fff",
                  "background": "transparent",
                  "border": "none",
                  "textAlign": "center",
                  "fontSize": "1.4rem",
                  "margin": "0 0 16px"
                }
              }
            },
            {
              "type": "Answer",
              "props": {
                "label": "Venus",
                "value": "venus"
              }
            },
            {
              "type": "Answer",
              "props": {
                "label": "Mars",
                "value": "mars"
              }
            },
            {
              "type": "Answer",
              "props": {
                "label": "Jupiter",
                "value": "jupiter"
              }
            },
            {
              "type": "Answer",
              "props": {
                "label": "Saturn",
                "value": "saturn"
              }
            }
          ]
        }
      }
    }
  }
}
```

In your host, record the question and clear last round's answers:

```
currentQuestion = { prompt: "...", correct: "mars" }
answers = {}
send member.update to all player ids with the payload above
```

Because each `Answer` button has `event: "answer"` and a `value`, a tap will arrive as a `msg` with that value.

## 2. Receive and confirm

When a player taps an answer you get a `msg`. Record it, score it, and — per [best practices](/advanced/best-practices) — immediately confirm receipt with a fresh screen so the player knows it landed:

```
on "msg" (payload):
  if payload.event != "answer": return
  id = payload.from
  choice = payload.message.value
  answers[id] = { choice, ms: payload.message.ms }   // ms = how fast they answered

  send member.update to [id]:
    header.text = "Locked in!"
    components = [ Text: "You answered **" + label(choice) + "**. Hang tight…" ]
```

Stashing `payload.message.ms` lets you award speed bonuses later — it's the milliseconds from the question appearing to the tap.

::callout{icon="i-lucide-clock"}
**Timed rounds?** Use the `change` event. `Choices` and other inputs emit `change` as the player edits, before they submit. Keep the latest `change.value` per player, and when your timer expires you can accept their in-progress selection even if they never tapped submit.
::

## 3. Reveal the result

Once everyone's answered (or your timer runs out), score the round and push each player a personalized result. Award points — say 1000 for correct, plus a speed bonus:

```
for id, answer in answers:
  correct = (answer.choice == currentQuestion.correct)
  if correct:
    points = 1000 + speedBonus(answer.ms)
    players[id].score += points

  send member.update to [id]:
    header.text = players[id].name + " — " + players[id].score + " pts"
    components = correct
      ? [ Text: "✅ Correct! **+" + points + "**" ]
      : [ Text: "❌ It was **Mars**." ]
```

Notice the header now shows the running score — small, constant feedback keeps players engaged.

## 4. Get ready for the next round

A brief interstitial sets the pace before you loop back to step 1:

```json
{
  "type": "member.update",
  "payload": {
    "to": ["aaaa...", "bbbb..."],
    "data": {
      "ui": {
        "header": { "text": "Get ready…" },
        "main": {
          "align": "center",
          "components": [
            {
              "type": "Text",
              "props": {
                "text": "Next question in 3…",
                "style": {
                  "color": "#fff",
                  "background": "transparent",
                  "border": "none",
                  "textAlign": "center",
                  "fontSize": "1.5rem"
                }
              }
            }
          ]
        }
      }
    }
  }
}
```

Then increment your question index and repeat from **step 1**. When you run out of questions, send a final standings screen instead.

## The loop, at a glance

1. **Ask** — push the question + answer buttons (`member.update`).
2. **Receive** — handle the `answer` `msg`; confirm receipt with a new screen.
3. **Reveal** — score it, push a personalized result with the running total.
4. **Get ready** — short interstitial, then back to step 1.

That four-beat rhythm — ask, confirm, reveal, ready — is the backbone of almost every Hackbox game. The [best practices](/advanced/best-practices) page digs into *why* each beat matters, and the [Playground](/tutorials/playground) lets you prototype these screens against your own phone before you write any host code.
